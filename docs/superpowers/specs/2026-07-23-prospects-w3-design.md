# W3 — Δυνητικοί Πελάτες · Newsletter · Opportunity — Design + Plan

**Date:** 2026-07-23 · Μετά τα W1 (`836e8e6`) + W2 (`dea1a76`). Ref: pb-ref `lib/programs/eligibility.ts` (per-company engine — το γυρνάμε **bulk** πάνω σε Trdr). **Approved** (user: «προχώρα και τα δύο w3, w4»).

## Σχεδίαση
1. **Matching (server-side)**: στη σελίδα προγράμματος, tab «Δυνητικοί πελάτες». Ο χρήστης επιλέγει ποια κριτήρια θα εφαρμοστούν (checkboxes): **Περιφέρεια** (ProgramRegion names ↔ Trdr.regionCode→Region climb σε level-3, normRegion bidirectional substring όπως ref), **Επιλέξιμοι ΚΑΔ** (ProgramKad + kadRule ALL_EXCEPT_LISTED/ONLY_LISTED/MIXED, hierarchical dotted-prefix match ↔ TrdrKad), **Νομική μορφή/τύπος** (ProgramEligibleLegalForm ↔ Trdr.appLegalForm μέσω canonicalLegalForm port). Bulk: όλα τα Trdr (ISACTIVE=1, SODTYPE 13), per-row pure `evaluateTrdrEligibility` (port ref: kadMatches, canonicalLegalForm, normRegion — μόνο τα 3 επιλεγμένα κριτήρια, τα λοιπά ref κριτήρια (ΕΜΕ/έτη) εκτός v1). Αποτέλεσμα: λίστα `{trdrId, name, email, matched:[criteria], failed:[criteria]}` — eligible μόνο όσα περνούν ΟΛΑ τα επιλεγμένα.
2. **`ProgramLead` model**: programId→Program Cascade, trdrId→Trdr Cascade, email, tokenHash @unique (sha256, raw once — pattern portal-token W2d), `status ProgramLeadStatus (PENDING|SENT|FAILED|CLICKED)` — **click = εκδήλωση ενδιαφέροντος = Ευκαιρία (opportunity)**, sentAt/clickedAt, @@unique([programId, trdrId]).
3. **Newsletter**: από τη λίστα eligible → επιλογή παραληπτών (checkboxes, default όσοι έχουν email) → «Αποστολή ενημέρωσης»: ανά παραλήπτη προσωποποιημένο email (Mailgun `sendMail`, escapeHtml): όνομα, τίτλος/σύνοψη προγράμματος, προθεσμία υποβολής, **tracked link** `${AUTH_URL}/go/{rawToken}`. Δημιουργεί/ενημερώνει ProgramLead (upsert programId+trdrId, νέο token σε επαναποστολή). Mailer-gated (isMailerConfigured), per-recipient try/catch counts.
4. **Public click route** `src/app/go/[token]/page.tsx` (force-dynamic, εκτός (app)): validate hash → mark CLICKED+clickedAt (μία φορά — repeat clicks δεν ξαναγράφουν) → self-contained ευχαριστήρια σελίδα (τίτλος προγράμματος + «Το ενδιαφέρον σας καταγράφηκε — θα επικοινωνήσουμε μαζί σας»). Invalid → γενικό «Ο σύνδεσμος δεν είναι έγκυρος». Κανένα άλλο data leak.
5. **Ευκαιρίες UI**: στο ίδιο tab, πίνακας leads (όνομα, email, status badge — CLICKED=«Ευκαιρία» coral/ok, sentAt/clickedAt) + κουμπί «Δημιουργία έργου» ανά CLICKED lead → υπάρχον `createApplication(trdrId, programId)` → link στο έργο.
6. **Gates**: matching/λίστα `programs.manage` (υπάρχον)· αποστολή `programs.manage`· public route μόνο token. ΚΑΝΕΝΑ νέο permission.

## Plan (tasks)
- **T1 Schema**: `ProgramLeadStatus` enum + `ProgramLead` + back-relations (Program.leads, Trdr.programLeads). Migration `prospects_w3`. TDD dmmf test.
- **T2 Pure engine**: `src/lib/prospects/eligibility.ts` — ports από ref: `kadMatches` (dotted hierarchical prefix), `evalKadRule(kadRule, programKads, trdrKads)`, `canonicalLegalForm`, `normRegion`/region name match, `evaluateTrdrEligibility(trdr, program, selectedCriteria)` → `{eligible, matched[], failed[]}`. TDD πλήρες (kadRule 4 modes, prefix match, legal forms, region substring).
- **T3 Actions + send + route**: `src/lib/prospects/actions.ts`: `findProspects(programId, criteria)` (bulk: load program kads/regions/legalForms + Trdr batch με TrdrKad+Region level-3 climb → pure eval → list)· `sendProgramNewsletter(programId, trdrIds[])` (mailer-gated, upsert lead + token, send, counts)· `listProgramLeads(programId)`· `createOpportunityApplication(leadId)` (→ υπάρχον createApplication + return appId). Public `src/lib/prospects/click.ts` `recordLeadClick(raw)` + route `/go/[token]`. TDD: guards + click idempotency + send counts (hoisted mocks).
- **T4 UI**: tab «Δυνητικοί πελάτες» στο program-editor (criteria checkboxes → «Αναζήτηση» → results table με matched/failed chips → επιλογή → «Αποστολή ενημέρωσης» confirm+counts toast) + section «Ευκαιρίες & Αποστολές» (leads table + «Δημιουργία έργου»). tsc/build/tests.
- **T5 Review+merge**: holistic (token hash-only, click idempotent+no-leak, bulk perf sanity ≤2 queries/batch, kadRule fidelity vs ref, gates) → fix → finish. Χωρίς sync-permissions.
